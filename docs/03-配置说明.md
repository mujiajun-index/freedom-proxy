# 配置说明

freedomProxy 的全部配置集中在一个 JSON 文件 `config.json` 中，部分敏感项可由环境变量覆盖。本文档说明配置文件结构、各字段含义、环境变量，以及 IP 白名单语法。

## 1. 配置文件总览

`config.json` 由服务在首次启动时自动生成（含随机 `adminToken`），之后由管理后台维护。完整示例：

```json
{
  "adminToken": "32dfa51e",
  "adminUser": "admin",
  "adminPasswordHash": "<bcrypt/sha256 哈希值>",
  "ipWhitelist": "192.168.1.10;10.0.0.0/24",
  "port": 3000,
  "sessionSecret": "<随机字符串，用于签发会话>",
  "accessLog": {
    "enabled": true,
    "file": "logs/access.jsonl",
    "logProxyOnly": false,
    "maxFileMB": 100,
    "keepFiles": 5,
    "retentionDays": 30
  },
  "mappings": [
    {
      "id": "map_openai",
      "prefix": "/openai",
      "target": "https://api.openai.com",
      "enabled": true,
      "note": "OpenAI 官方接口"
    },
    {
      "id": "map_internal",
      "prefix": "/internal",
      "target": "https://10.0.0.5:9000",
      "enabled": true,
      "note": "内网服务"
    }
  ]
}
```

## 2. 字段说明

### 顶层字段

| 字段 | 类型 | 默认 | 说明 |
| --- | --- | --- | --- |
| `adminToken` | string | 首启随机生成 | 管理后台地址 token，访问 `/{adminToken}` 进入后台；固定不变 |
| `adminUser` | string | `admin` | 管理员账号 |
| `adminPasswordHash` | string | — | 管理员密码哈希；可被环境变量 `ADMIN_PASSWORD` 覆盖 |
| `ipWhitelist` | string | `""` | IP 白名单，`;` 分隔，支持单 IP / CIDR；空 = 不限制 |
| `port` | number | `3000` | Node 服务监听端口 |
| `sessionSecret` | string | 首启随机生成 | 会话签名密钥，泄露会导致会话被伪造 |
| `accessLog` | object | 见下 | 访问日志配置，详见 [accessLog](#accesslog-访问日志) |
| `mappings` | array | `[]` | 代理映射规则数组，见下表 |

### mappings（映射规则）

数组成员字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | string | 是 | 唯一标识 |
| `prefix` | string | 是 | 本地路径前缀，以 `/` 开头 |
| `target` | string | 是 | 目标基础 URL |
| `enabled` | boolean | 否 | 是否启用，默认 `true` |
| `note` | string | 否 | 备注 |

## 3. 环境变量

环境变量优先级高于 `config.json`，便于在不改动配置文件、不入库敏感信息的前提下覆盖：

| 环境变量 | 覆盖字段 | 说明 |
| --- | --- | --- |
| `PORT` | `port` | 监听端口 |
| `ADMIN_USER` | `adminUser` | 管理员账号 |
| `ADMIN_PASSWORD` | `adminPasswordHash` | 管理员密码明文，服务启动时自行哈希；设置后以明文校验，无需手动哈希 |
| `CONFIG_PATH` | — | `config.json` 的存放路径，默认项目根目录 |
| `TRUST_PROXY` | — | 是否信任 nginx 转发的 `X-Real-IP`/`X-Forwarded-For`（`true`/`false`），影响 IP 白名单取值 |
| `SESSION_SECRET` | `sessionSecret` | 会话签名密钥 |
| `ACCESS_LOG_PATH` | `accessLog.file` | 访问日志文件路径，覆盖配置中的 `accessLog.file` |

> 生产建议：`ADMIN_PASSWORD`、`SESSION_SECRET` 通过环境变量注入，`config.json` 仅存映射规则与白名单等非敏感项。

## 4. 重点字段详解

### adminToken（管理后台地址）

- 首次启动若为空，则随机生成（建议 8 位以上十六进制）。
- 生成后写入 `config.json`，**之后固定不变**，重启保持一致。
- 获取方式：启动日志打印，或直接读 `config.json`。
- 重置方式：清空该字段后重启，即重新生成（旧地址立即失效）。

### 管理员凭据

- `adminUser` + `adminPasswordHash`（或 `ADMIN_PASSWORD` 明文）用于登录校验。
- 密码建议使用 bcrypt 等带盐哈希；若用 `ADMIN_PASSWORD` 环境变量，服务内部完成哈希/比对，配置文件可不出现明文。
- 登录接口见 [API 文档 · 登录](04-API文档.md#登录)。

### ipWhitelist（IP 白名单）

- **格式**：字符串，多个条目用英文分号 `;` 分隔。
- **支持的条目**：
  - 单 IPv4：`192.168.1.10`
  - IPv4 CIDR：`10.0.0.0/24`
  - 单 IPv6：`2001:db8::1`
  - IPv6 CIDR：`2001:db8::/32`
- **空值语义**：`""` 或字段缺省 = **不限制**，所有 IP 可访问。
- **匹配规则**：客户端 IP 落在任一条目内即放行；否则返回 `403`。
- **取值来源**：当 `TRUST_PROXY=true` 时优先取 `X-Real-IP`/`X-Forwarded-For` 的最右可信值；否则取 TCP 连接对端 IP。
- **示例**：
  - 仅本机：`127.0.0.1;::1`
  - 一个网段 + 一个公网 IP：`10.0.0.0/24;203.0.113.5`
  - 不限制：`""`

> 也可在 nginx 层用 `allow`/`deny` 或 `geo` 实现同等效果（推荐，效率更高），见 [部署指南](05-部署指南.md#3-ip-白名单落地)。建议两层配置保持一致。

### accessLog（访问日志）

访问日志写入**独立的 JSONL 文件**（每行一条 JSON），与 `config.json` 分离，避免高频追加数据撑大配置文件。各子字段：

| 子字段 | 类型 | 默认 | 说明 |
| --- | --- | --- | --- |
| `enabled` | boolean | `true` | 是否开启访问日志 |
| `file` | string | `logs/access.jsonl` | 日志文件路径（相对工作目录或绝对路径） |
| `logProxyOnly` | boolean | `false` | `true`=仅记录代理转发请求；`false`=记录代理与其它外部访问（管理后台自身请求始终不记录） |
| `maxFileMB` | number | `100` | 单文件大小上限，达到后轮转切分 |
| `keepFiles` | number | `5` | 保留的历史轮转文件数 |
| `retentionDays` | number | `30` | 日志保留天数，超期自动清理 |

每条日志字段：`time`、`ip`、`method`、`path`、`status`、`elapsedMs`、`mapping`、`target`、`userAgent`、`bytes`，含义见 [功能说明 · 访问日志](02-功能说明.md#5-访问日志)。

> 取真实客户端 IP 依赖 `TRUST_PROXY`：经 nginx 反代时设为 `true`，否则日志里的 `ip` 会是 nginx/上游的内网地址。

## 5. 配置变更生效

- **映射增删改 / 白名单修改**：管理后台保存后**立即热生效**，无需重启。
- **端口 / token / 凭据 / sessionSecret 变更**：需重启服务（token/sessionSecret 变更会使现有会话失效）。
- **访问日志配置变更**：`accessLog.enabled`/`file`/轮转策略变更建议重启服务以稳妥应用；日志查询与清空接口可热生效。
- 所有写入均为对 `config.json` 的原子写（先写临时文件再重命名），避免并发损坏。
