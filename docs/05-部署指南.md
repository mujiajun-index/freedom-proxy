# 部署指南

本文档说明如何运行 freedomProxy 的 Node.js 服务、用 nginx 反向代理对外提供服务、在 nginx 与应用层落地 IP 白名单，以及使用 PM2/systemd 守护进程与配置 HTTPS。

整体拓扑：**客户端 → nginx（80/443，HTTPS 终止 + 反代 + 白名单）→ Node.js（127.0.0.1:3000，代理引擎 + 后台 + API）**。

---

## 1. 运行 Node.js 服务

### 1.1 环境要求

- Node.js ≥ 18（内置 `fetch`、原生 HTTP；TS 经编译或 `tsx`/`ts-node` 运行）
- npm / pnpm / yarn 任一

### 1.2 安装与启动

```bash
# 安装依赖
npm install

# 方式 A：开发期直接用 tsx 运行 TS
npx tsx src/server/index.ts

# 方式 B：编译为 JS 后运行
npm run build        # tsc -> dist/
node dist/server/index.js
```

首次启动会：

1. 在 `CONFIG_PATH`（默认项目根）下生成 `config.json`；
2. 随机生成 `adminToken` 与 `sessionSecret` 并写入；
3. 在控制台打印管理后台地址，例如：
   ```
   freedomProxy 已启动，监听端口 3000
   管理后台地址: http://localhost:3000/32dfa51e
   ```

> 记下该地址（也可随时查看 `config.json` 的 `adminToken`）。`adminToken` 固定不变，重启后保持一致。

### 1.3 通过环境变量覆盖配置

```bash
# 示例：自定义端口、注入管理员密码、开启信任前置代理
PORT=3000 \
ADMIN_PASSWORD='你的强密码' \
SESSION_SECRET='一段随机长字符串' \
TRUST_PROXY=true \
node dist/server/index.js
```

环境变量优先级高于 `config.json`，详见 [配置说明 · 环境变量](03-配置说明.md#3-环境变量)。

## 2. nginx 反向代理

nginx 监听对外端口（80/443），将请求转发到本机 Node 服务。

```nginx
# /etc/nginx/conf.d/freedomproxy.conf

upstream freedomproxy {
    server 127.0.0.1:3000;
    keepalive 32;
}

server {
    listen 443 ssl http2;
    server_name proxy.example.com;

    # --- HTTPS 证书（见第 5 节） ---
    ssl_certificate     /etc/letsencrypt/live/proxy.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/proxy.example.com/privkey.pem;

    # --- 透传真实客户端 IP，供应用层白名单使用 ---
    proxy_set_header Host              $host;
    proxy_set_header X-Real-IP         $remote_addr;
    proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    # --- 代理转发相关：支持长连接、流式（SSE/大文件）、超时 ---
    proxy_http_version 1.1;
    proxy_set_header   Connection "";
    proxy_buffering    off;          # 流式响应不缓冲
    proxy_read_timeout 300s;
    proxy_send_timeout 300s;

    # 所有路径统一交给 Node 处理（含 /{token} 后台与映射前缀）
    location / {
        proxy_pass http://freedomproxy;
    }
}

# 80 跳转到 HTTPS
server {
    listen 80;
    server_name proxy.example.com;
    return 301 https://$host$request_uri;
}
```

要点：

- **随机管理地址无需 nginx 特判**：`/{token}` 由 Node 处理，nginx 只需 `location /` 全量反代。
- **`proxy_buffering off`**：保证 SSE / 流式响应实时回传。
- **`X-Real-IP` / `X-Forwarded-For`**：配合应用层 `TRUST_PROXY=true`，使 IP 白名单能取到真实客户端 IP。
- 重载配置：`sudo nginx -t && sudo nginx -s reload`。

## 3. IP 白名单落地

IP 白名单可在 **nginx 层（推荐）** 或 **应用层** 实现，二者描述一致即可。详见 [功能说明 · IP 白名单](02-功能说明.md#4-ip-白名单)。

### 3.1 nginx 层（推荐）

**方式 A：`allow` / `deny`（适合少量单 IP）**

```nginx
location / {
    allow 192.168.1.10;
    allow 203.0.113.5;
    allow 10.0.0.0/24;      # 支持网段
    deny  all;
    proxy_pass http://freedomproxy;
}
```

**方式 B：`geo` + 变量（适合 CIDR 较多、可维护）**

```nginx
geo $allowed {
    default        0;
    192.168.1.10   1;
    203.0.113.5    1;
    10.0.0.0/24    1;
}

server {
    # ...
    location / {
        if ($allowed = 0) {
            return 403;
        }
        proxy_pass http://freedomproxy;
    }
}
```

> nginx 原生支持 IPv4/IPv6 单 IP 与 CIDR，与配置文件 `ipWhitelist` 的语法（`;` 分隔）一一对应，只需在部署时同步两处。

### 3.2 应用层（兜底）

当无法修改 nginx（如托管环境）或服务直接暴露时，由 Node 中间件读取 `config.json` 的 `ipWhitelist` 进行校验：

- `TRUST_PROXY=true`：取 `X-Real-IP` / `X-Forwarded-For` 最右可信值。
- `TRUST_PROXY=false`：取 TCP 对端 IP（服务直接对外时用此值）。
- 未命中 → 返回 `403`。

> 建议 nginx 与应用层白名单保持一致；任一层生效即可拦截。修改后应用层**热生效**，nginx 层需 `nginx -s reload`。

## 4. 进程守护

### 4.1 PM2

```bash
npm install -g pm2

# 方式 A：直接守护编译产物
pm2 start dist/server/index.js --name freedomproxy

# 方式 B：使用 ecosystem 配置（推荐，便于注入环境变量）
pm2 start ecosystem.config.js
pm2 save
pm2 startup        # 开机自启
```

`ecosystem.config.js` 示例：

```js
module.exports = {
  apps: [{
    name: 'freedomproxy',
    script: 'dist/server/index.js',
    instances: 1,            // 配置文件存储，建议单实例避免写冲突
    env: {
      NODE_ENV: 'production',
      PORT: 3000,
      ADMIN_PASSWORD: '你的强密码',
      SESSION_SECRET: '一段随机长字符串',
      TRUST_PROXY: 'true'
    }
  }]
};
```

> 因采用单 `config.json` 文件存储，建议**单实例**运行；如需多实例，需引入文件锁或将存储替换为数据库（超出本项目范围）。

### 4.2 systemd

`/etc/systemd/system/freedomproxy.service`：

```ini
[Unit]
Description=freedomProxy
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/freedomproxy
EnvironmentFile=/opt/freedomproxy/.env
ExecStart=/usr/bin/node /opt/freedomproxy/dist/server/index.js
Restart=on-failure
RestartSec=3

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now freedomproxy
sudo systemctl status freedomproxy
```

`.env` 文件示例（`EnvironmentFile` 不支持 `export`，仅 `KEY=VALUE`）：

```
PORT=3000
ADMIN_PASSWORD=你的强密码
SESSION_SECRET=一段随机长字符串
TRUST_PROXY=true
```

## 5. HTTPS（证书）

推荐 Let's Encrypt + certbot：

```bash
sudo certbot --nginx -d proxy.example.com
```

certbot 会自动修改 nginx 配置写入证书路径并开启 80→443 跳转，证书到期自动续期。

如使用自签证书或云厂商证书，将 `ssl_certificate` / `ssl_certificate_key` 指向对应文件即可。

## 6. 部署检查清单

- [ ] Node 服务已启动，控制台打印了管理后台 `/{token}` 地址
- [ ] `config.json` 已生成，`adminToken`、`sessionSecret` 已写入
- [ ] 管理员密码已通过 `ADMIN_PASSWORD` 注入（避免弱默认值）
- [ ] nginx 反代生效，`https://你的域名/{token}` 可打开登录页
- [ ] `X-Real-IP` / `X-Forwarded-For` 已透传，应用层 `TRUST_PROXY=true`
- [ ] IP 白名单已在 nginx 或应用层配置（或确认为「不限制」）
- [ ] 进程守护（PM2/systemd）与开机自启已配置
- [ ] HTTPS 已启用，80 已跳转 443
- [ ] 新增一条映射并经域名验证转发正确（含查询参数、POST body、流式响应）
