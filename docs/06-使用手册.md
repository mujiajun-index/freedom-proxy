# 使用手册

面向管理员的操作指引：从拿到随机管理地址、首次登录，到新增代理映射、配置 IP 白名单、验证转发。术语与字段对应 [配置说明](03-配置说明.md)，接口细节见 [API 文档](04-API文档.md)。

---

## 0. 前置准备

确认服务已按 [部署指南](05-部署指南.md) 启动，并已记录：
- 服务对外地址（如 `https://proxy.example.com`）
- 管理员账号与密码（账号默认 `admin`，密码由部署时 `ADMIN_PASSWORD` 注入或配置文件设定）

---

## 1. 获取管理后台地址

管理后台位于一个**随机地址**下（类似宝塔面板的安全入口），形如：

```
https://proxy.example.com/32dfa51e
                主机名  /  token
```

获取 token（`adminToken`）的方式：

1. **启动日志**：服务首次启动会在控制台打印 `管理后台地址: ...`。
2. **配置文件**：查看 `config.json` 中的 `adminToken` 字段。

> token 首次生成后**固定不变**。直接访问根路径 `/` 或其它非映射路径都不会暴露后台。

---

## 2. 首次登录

1. 浏览器打开 `https://proxy.example.com/{token}`，进入登录页。
2. 输入管理员账号与密码，提交。
3. 登录成功后进入后台首页（映射列表）。

若提示「用户名或密码错误」：
- 确认账号是否为部署设定的 `adminUser`（默认 `admin`）。
- 密码以部署时 `ADMIN_PASSWORD` 为准；如需重置，更新该环境变量后重启服务。

---

## 3. 管理代理映射

### 3.1 新增映射

1. 进入「代理映射」页面，点击「新增」。
2. 填写：
   - **路径前缀**：如 `/openai`（必须以 `/` 开头）。
   - **目标地址**：如 `https://api.openai.com`。
   - **启用**：勾选（默认启用）。
   - **备注**：可选，如「OpenAI 官方接口」。
3. 保存，规则**立即生效**，无需重启。

### 3.2 验证转发

新增 `/openai → https://api.openai.com` 后：

```
请求：  curl https://proxy.example.com/openai/v1/models
转发：  curl https://api.openai.com/v1/models
```

也可用后台「连通性测试」按钮快速探测目标是否可达（返回状态码与耗时）。

### 3.3 编辑 / 禁用 / 删除

- **编辑**：修改前缀、目标、备注后保存。
- **禁用**：将「启用」关闭，该前缀暂时不再转发（保留配置）。
- **删除**：移除该条规则。

> 匹配遵循「最长前缀优先」：若同时存在 `/api` 与 `/api/v2`，请求 `/api/v2/x` 会命中 `/api/v2`。

---

## 4. 配置 IP 白名单

### 4.1 何时需要

当你希望**只有指定 IP/网段**才能访问本服务（含代理与后台）时启用。

### 4.2 配置步骤

1. 进入「IP 白名单」页面。
2. 在输入框填写允许的来源，**多个用 `;` 分隔**，支持单 IP 与 CIDR，例如：
   ```
   192.168.1.10;10.0.0.0/24;203.0.113.5
   ```
3. 保存，**立即热生效**。
4. **留空**则表示「不限制」（所有 IP 可访问）。

### 4.3 注意事项

- ⚠️ **保存前确认自身 IP 在白名单内**，否则会被立即拦截、无法再访问后台。
- 若不慎被拦截：
  - 直接编辑 `config.json` 的 `ipWhitelist`（加入自身 IP 或置空）后重启服务；
  - 或在 nginx 层调整 `allow`/`deny`（若白名单落在 nginx）。
- 取真实客户端 IP 依赖 `TRUST_PROXY`：经 nginx 反代时需设为 `true`，服务直接暴露时设为 `false`。详见 [部署指南 · IP 白名单落地](05-部署指南.md#3-ip-白名单落地)。

---

## 5. 重置 / 更换管理地址

`adminToken` 固定不变，但可主动重置：

1. 停止服务。
2. 清空 `config.json` 中的 `adminToken`（或删除整个字段）。
3. 重新启动，服务生成新 token 并写入。
4. **旧地址立即失效**，需用日志/配置文件中的新地址访问。

---

## 6. 常见排查

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 忘记管理地址 | token 固定但未记录 | 查 `config.json` 的 `adminToken` |
| 后台打不开（404/提示页） | 地址拼错、token 失效 | 核对 token；必要时按第 5 节重置 |
| 登录提示密码错误 | 密码与部署设定不一致 | 核对/重置 `ADMIN_PASSWORD` 后重启 |
| 所有请求 403 | 自身 IP 未在白名单 | 编辑 `config.json` 或 nginx 白名单加入自身 IP |
| 转发返回 502 | 目标不可达/DNS/超时/TLS 错误 | 用「连通性测试」排查；检查目标地址与网络 |
| 转发 404 | 请求路径未命中任何启用映射 | 核对 `prefix` 拼写与 `enabled` 状态 |
| 经 nginx 后白名单取到内网 IP | 未透传真实 IP 或 `TRUST_PROXY` 未开 | nginx 设 `X-Real-IP`/`X-Forwarded-For`，应用设 `TRUST_PROXY=true` |
| 流式响应（SSE）不实时 | nginx 缓冲 | nginx 设 `proxy_buffering off;` |
| 修改端口/token 后未生效 | 部分配置需重启 | 重启 Node 服务（参考 [部署指南](05-部署指南.md#4-进程守护)） |
